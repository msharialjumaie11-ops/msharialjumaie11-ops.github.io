üóÉÔ∏è What Is a Database?

A database is like a digital filing cabinet‚Äîit stores information in a structured way so you can easily retrieve and manipulate it. Instead of one giant spreadsheet, databases use tables, which are made up of:

  • Rows: Each row represents a single record (e.g., one student).
  • Columns: Each column represents a specific attribute (e.g., name, age, grade).

For example, a table called Students might look like this:

ID Name Age Grade
1 Ahmad 20 A
2 Khalid 22 B

üß† Why Use a Database?

  • Scalability: Can handle millions of records.
  • Speed: Fast data retrieval using indexes and queries.
  • Security: Controlled access to sensitive data.
  • Multi-user access: Many users can interact with the data simultaneously.

üßæ What Is SQL?

SQL (Structured Query Language) is the standard language used to communicate with relational databases. It allows you to:

  • Create tables and structures
  • Insert new data
  • Query existing data
  • Update records
  • Delete data

These operations are often referred to as CRUD:

  • Create
  • Read
  • Update
  • Delete

üìö SQL Structure Overview

SQL (Structured Query Language) is the standard language used to interact with relational databases. Its structure is built around clear commands like SELECT, INSERT, UPDATE, and DELETE, each serving a specific purpose in managing data.

SQL is composed of statements that follow a consistent pattern. Each statement begins with a keyword (like SELECT or INSERT) and is followed by clauses that define what data to work with and how.

üü¢ SELECT ‚Äì Retrieve Data

The SELECT statement is used to retrieve data from one or more columns in a table.

SELECT column1, column2 FROM table_name WHERE condition;

üß± Creating Table in SQL

A table is a structured collection of data organized into rows and columns:

  • Rows: Represent individual records (e.g., one student).
  • Columns: Represent attributes of those records (e.g., name, age, grade).

üßæ Basic Syntax of CREATE TABLE

CREATE TABLE table_name (
  column1 datatype,
  column2 datatype,
  column3 datatype,
  ...
);
  • table_name: The name of your table.
  • column1, column2, etc.: The names of the columns.
  • datatype: Specifies the type of data each column can store (e.g., INT, VARCHAR, DATE).

üß™ Example: Creating a Students Table

CREATE TABLE Students (
  ID INT,
  Name VARCHAR(50),
  Age INT,
  Grade CHAR(1)
);

This creates a table named Students with:

  • ID: An integer for student ID
  • Name: A string up to 50 characters
  • Age: An integer
  • Grade: A single character (e.g., A, B, C)

üõ°Ô∏è Adding Constraints

Constraints help enforce rules on the data:

  • PRIMARY KEY: Uniquely identifies each row
  • NOT NULL: Prevents empty values
  • UNIQUE: Ensures all values in a column are different

Example with constraints:

CREATE TABLE Students (
  ID INT PRIMARY KEY,
  Name VARCHAR(50) NOT NULL,
  Age INT CHECK (Age >= 0),
  Grade CHAR(1)
);

üßæ Inserting Data into a Table

To insert data into a table, you first need to create the table with defined columns and data types. Then you use the INSERT INTO command to add records. This process is essential for building and populating any relational database.

üß± Step 1: Create a Table

Start by defining the structure of your table using the CREATE TABLE statement. Here's an example for a simple Students table:

CREATE TABLE Students (
  ID INT PRIMARY KEY,
  Name VARCHAR(50),
  Age INT,
  Grade CHAR(1)
);
  • ID: A unique identifier for each student.
  • Name: A string up to 50 characters.
  • Age: An integer value.
  • Grade: A single character (e.g., A, B, C).

üü¢ Step 2: Insert Data into the Table

Use the INSERT INTO statement to add records:

INSERT INTO Students (ID, Name, Age, Grade)
VALUES (1, 'Aisha', 20, 'A');

You can insert multiple records like this:

INSERT INTO Students (ID, Name, Age, Grade)
VALUES 
(2, 'Khalid', 22, 'B'),
(3, 'Sara', 19, 'A');

üß† Notes on INSERT

  • You must match the number and order of values with the specified columns.
  • If a column is auto-incremented, you can skip it in the INSERT statement.
  • Use NULL for unknown or optional values.
